CONNECTION MADE SIMPLE — CHANGELOG
===================================
All changes since the project was first uploaded to GitHub (2026-06-12).
Original codebase was copied from the youth-allocation-platform project.

PHASE 3: VERCEL DEPLOYMENT  (2026-06-12)
-------------------------------------------------------------------------------
- Added Vercel serverless handler (api/index.ts) and vercel.json configuration
- Extensive debugging and fixes for ESM/CJS module format compatibility with
  @vercel/node, ultimately settling on CommonJS output + @vercel/node bundling
- Routes only API paths to the Lambda; static assets (HTML/CSS/JS) served by
  Vercel CDN for instant load
- Added .vercelignore and .env.example
- Replaced in-memory token store with stateless HMAC-signed JWT sessions —
  tokens survive across serverless invocations
- Fixed Lambda crashes: unhandledRejection handler, postgres connection
  timeouts (idle_timeout, connect_timeout, max_lifetime)
- Set Lambda maxDuration to 30s to accommodate cold starts
- Pinned function to syd1 region to co-locate with Supabase Sydney
- Raised DB connection pool from max:1 to max:5, eliminating head-of-line
  blocking that caused app-wide freezes during slow queries or imports
- Added import upload timeout of 90s (separate from the 20s default)
- Added fetch AbortController (20s) and Postgres statement_timeout (15s)
- Added request coalescing in API.get() to deduplicate concurrent in-flight
  requests for the same endpoint

-------------------------------------------------------------------------------
PHASE 4: SECURITY HARDENING  (2026-06-12 to 2026-06-13)
-------------------------------------------------------------------------------
- Replaced SHA-256+salt password hashing with bcryptjs (12 rounds)
- Transparent password upgrade: legacy hashes re-hashed to bcrypt on first
  successful login — no user action required
- Added Row-Level Security (RLS) on all 13 Supabase tables (migration 006);
  anon key connections denied, DATABASE_URL bypasses RLS
- Added login rate limiting: 10 attempts per IP per 15-minute window,
  returning 429 + Retry-After header
- Added HTTP security headers: X-Content-Type-Options, X-Frame-Options,
  Referrer-Policy, Permissions-Policy
- Used timingSafeEqual for password comparison to prevent timing-based oracles
- Error sanitization in production (internals not leaked to clients)
- Added CORS_ORIGINS wildcard warning on startup in production
- Added SECURITY-ACTIONS.txt listing manual operator security steps
- Corrected admin password hash in seed migration (002_seed_admin.sql)
- Settings singleton: atomic upsert prevents duplicate rows on concurrent
  cold starts; enforced via migration 003

PHASE 6: DATA IMPORT  (2026-06-13)
-------------------------------------------------------------------------------
Service Import:
- Real Elvanto/UCare CSV parsing: normalises column names (First Name to
  first_name, School Grade to grade), skips Attended/Absent summary columns,
  extracts dates from descriptive session headers (e.g. "Brisbane - YS - Fri 30/01")
- Replace semantics: each import is authoritative (full-year export); clears
  prior sessions/attendance, keeps all students and connections, resets svcAttended
  to 0 for students absent from the new file
- Valid service model (migration 008): a session only counts if whole-ministry
  attendance >= configurable floor (serviceMinAttendance, default 100);
  holiday/term-break weeks excluded automatically; avg = attended divided by valid services
- Normalises date_of_birth from DD/MM/YYYY to ISO YYYY-MM-DD before DB write
- Deduplicates students keyed by ID (last CSV row wins); deduplicates attendance
  records keyed by studentId:sessionId — prevents ON CONFLICT errors from
  duplicate CSV rows
- Settings UI updated: "Valid Session Threshold %" replaced by "Min attendance
  for a valid service"

Group Import:
- New POST /import/group-csv endpoint parses Elvanto Group Individual Attendance
  CSVs: group header rows, "Last, First" name format, DD/MM/YYYY date columns,
  Y/N/- membership values
- Monday-week bucketing: attendance keyed to the Monday of each week, so a group
  meeting twice in one week counts once; per-student grp = weeks attended divided by
  weeks the group ran
- 0-attendance members (listed but never attended) are not counted as group members
- Replace semantics: clears prior lifegroup data, keeps students and connections,
  resets grp to 0 for students not in the new file
- Leader row detection: roll entries containing "(leader)", "(leaders)",
  "(assistant leader)", or "(assistant leaders)" auto-create/update Leader records
  (grade + gender from group name) and are excluded from youth attendance
- A leader appearing in multiple grade groups gets all grades assigned (grades
  unioned; matched by name rather than duplicated)
- Import tile split into two separate tiles: service and group

Performance:
- Batch all import saves with Promise.all; saveMany() bulk INSERT replaces
  N sequential round trips — total DB calls approximately 5 regardless of import size
- Chunked bulk inserts at 1,000 rows to stay within Postgres bind-param limit
- Per-group week IDs prevent PK collisions for students in multiple groups
- Import client timeout extended to 90s
- TRUNCATE ... CASCADE for deleteAll() replaces row-by-row DELETE (avoids
  15s statement_timeout on full reset)

SUMMARY OF CUMULATIVE CHANGES
-------------------------------------------------------------------------------
The project has evolved from a copy of youth-allocation-platform into a
production PWA for connection ministry management. Key milestones:

  Persistent backend   — Supabase (PostgreSQL) with full schema, RLS, and
                         migrations; Vercel serverless in the Sydney region
  Security             — bcrypt, HMAC sessions, rate limiting, security
                         headers, Row-Level Security
  Data ingestion       — Elvanto/UCare CSV import for service attendance and
                         lifegroup attendance; replace semantics;
                         valid-service thresholding
  Analytics            — term-aware trends (current + previous), lifegroup
                         stats endpoint, valid-services denominator for all
                         averages
  RBAC                 — gender-scoped grade logins, quad scoping,
                         cross-grade connect rules
  UX                   — SVG icons, PWA (manifest + service worker),
                         persistent shell, client-side cache, background
                         prefetch, pre-rendered dropdowns
  At-risk              — dynamic qualifiers (rising/declining/stopped/mixed),
                         tap-to-call, rebuilt collapsible sections
  Connection Audit     — per-lifegroup health, per-quad funnels,
                         executive deck
  Admin                — Clear Service/Group Data action, editable emails,
                         allocation import/export CSV round-trip
